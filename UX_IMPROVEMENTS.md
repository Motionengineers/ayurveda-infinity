# UX Principles Applied to AYURVEDA Website

## Key Principles to Implement

### 1. **Aesthetic Usability Effect**
- Beautiful design = perceived usability
- Apply: Clean, minimal design with the 60-30-10 palette

### 2. **Doherty Threshold** (<400ms response)
- Instant feedback for interactions
- Apply: Add loading states, hover effects, smooth transitions

### 3. **Fitts' Law** (Target size & distance)
- Make clickable areas large and close
- Apply: Larger buttons, cards with adequate padding

### 4. **Hick's Law** (Reduce choices)
- Fewer options = faster decisions
- Apply: Limit navigation items, simplify product filters

### 5. **Jakob's Law** (Familiar patterns)
- Follow web conventions
- Apply: Standard navigation, familiar e-commerce patterns

### 6. **Law of Common Region** (Grouping)
- Visually group related items
- Apply: Clear sections, card groups, related products

### 7. **Law of Prägnanz** (Simplicity)
- Simple, clear designs
- Apply: Minimal UI, clear hierarchy

### 8. **Law of Proximity** (Related items close)
- Place related elements together
- Apply: Related content grouped, consistent spacing

### 9. **Law of Similarity** (Similar items look alike)
- Visual consistency
- Apply: Consistent button styles, card designs

### 10. **Law of Uniform Connectedness** (Visual connections)
- Connect related items visually
- Apply: Lines, borders, backgrounds to show relationships

### 11. **Miller's Law** (7±2 items)
- Limit items per group
- Apply: Max 5-7 nav items, 3-6 products per row

### 12. **Occam's Razor** (Simplicity)
- Simplest solution is best
- Apply: Remove unnecessary elements

### 13. **Pareto Principle** (80/20)
- Focus on high-impact features
- Apply: Prioritize core flows (shop, quiz, rituals)

### 14. **Serial Position Effect** (First & last remembered)
- Put important items at start/end
- Apply: Hero CTA prominent, footer CTAs

### 15. **Von Restorff Effect** (Isolation)
- Make important items stand out
- Apply: Accent color for CTAs, contrast for key actions

### 16. **Zeigarnik Effect** (Progress indicators)
- Show progress for incomplete tasks
- Apply: Quiz progress, cart steps, checkout flow

### 17. **Postel's Law** (Be liberal, conservative)
- Accept varied input, provide consistent output
- Apply: Flexible form inputs, clear validation

### 18. **Tesler's Law** (Accept complexity)
- Don't pass complexity to users
- Apply: Handle backend complexity, simple UI

### 19. **Parkinson's Law** (Time limits)
- Set clear limits
- Apply: Limited-time offers, clear deadlines
